# Projecte Web Corporativa – FoodLogistic S.A.

## Introducció al cas

Una de les primeres inquietuds que FoodLogistic S.A. ens ha traslladat és la necessitat de millorar la seva presència a Internet. L’actual pàgina web es troba estètica i tècnicament desactualitzada i, a més, no compleix amb la normativa vigent en matèria de protecció de dades i serveis digitals (LOPDGDD i LSSI-CE).

Davant d’aquesta situació, el client sol·licita una proposta de renovació de la seva identitat digital, però també una **prova funcional real**. Per aquest motiu, el projecte inclou el desplegament del contingut en un servidor públic amb l’objectiu de mostrar com serà la nova web corporativa abans de la seva publicació definitiva.

Aquesta activitat forma part del mòdul i permet posar en pràctica un flux de treball professional basat en control de versions, documentació i analítica web.

---

## Objectius del projecte

- **Publicació professional:** desplegament de la web mitjançant GitHub Pages sota una URL pública.
- **Documentació del projecte:** elaboració d’un fitxer README.md com a carta de presentació del projecte i del seu autor.
- **Control i analítica:** integració d’un sistema de mesura de visites mitjançant StatCounter.

---

## Solució realitzada

### Estructura i desplegament

El projecte s’ha desenvolupat en local i s’ha publicat utilitzant **GitHub Pages**, complint amb l’estructura requerida:
- Carpeta `/docs` com a directori principal del desplegament.
- Fitxer `index.html` i recursos associats dins d’aquesta carpeta.
- Ús de la branca `main` per a la publicació, realitzant commits només quan hi ha versions estables.

### Analítica web (StatCounter)

S’ha implementat un **comptador invisible amb StatCounter** que permet:
- Mesurar el nombre de visites
- Analitzar la procedència dels usuaris
- Observar l’evolució del trànsit web

Aquesta informació és clau per poder millorar el contingut i el rendiment de la web en futures iteracions.
